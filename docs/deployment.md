# Deployment

> Living runbook for the uniapp blog at `blog.multilab.cc`.
> Pair this with `SPEC.md` (decisions/risks) and `tasks/todo.md` (status).

## Architecture in 30 seconds

```
[reader] -> https://blog.multilab.cc -> Cloudflare edge -> tunnel ->
            cloudflared (LocalSystem service) -> http://127.0.0.1:48080 ->
            Caddy (LocalSystem service) -> ./site/ (static files)
```

`./site/` is regenerated by `scripts/publish.ps1` from
`frontend-uniapp/dist/build/h5/`. Both `cloudflared` and `caddy` run as
NSSM-managed Windows services so the site survives reboots without
anyone logging in.

## One-time activation (per-machine, ~10 minutes)

Run from the repo root in PowerShell. Steps 1, 5, 7 are scripted; 2-4
and 6 are interactive (auth + UAC + reboot test).

### Step 1 - Install the three binaries (idempotent)

```powershell
powershell -ExecutionPolicy Bypass -File scripts/install-binaries.ps1
```

Drops `tools/caddy.exe`, `tools/cloudflared.exe`, `tools/nssm.exe`. All
gitignored.

### Step 2 - Cloudflare account auth (interactive, browser)

```powershell
tools\cloudflared.exe tunnel login
```

Opens a browser; pick the `multilab.cc` zone. Cert lands in
`%USERPROFILE%\.cloudflared\cert.pem`.

### Step 3 - Create the named tunnel

```powershell
tools\cloudflared.exe tunnel create blog
```

Writes credentials JSON to `%USERPROFILE%\.cloudflared\<UUID>.json`.
Note the UUID for the next step.

### Step 4 - Fill the cloudflared config

```powershell
copy configs\cloudflared.yml.sample configs\cloudflared.yml
notepad configs\cloudflared.yml
```

Replace `<YOUR_TUNNEL_UUID>` and `<YOUR_USERNAME>`. The credentials path
must be absolute (LocalSystem can't expand `%USERPROFILE%`).

### Step 5 - Bind the public hostname

```powershell
tools\cloudflared.exe tunnel route dns blog blog.multilab.cc
```

Writes a CNAME `blog.multilab.cc -> <UUID>.cfargotunnel.com` into the
Cloudflare DNS zone.

### Step 6 - Smoke-test interactively (two terminals)

```powershell
# Terminal A
powershell -ExecutionPolicy Bypass -File scripts/publish.ps1
tools\caddy.exe run --config configs\Caddyfile

# Terminal B
tools\cloudflared.exe --config configs\cloudflared.yml tunnel run blog

# Browser
# http://127.0.0.1:48080/         (local Caddy)
# https://blog.multilab.cc/       (via tunnel)
# Both pages should be identical.
```

Stop both with Ctrl-C once verified.

### Step 7 - Install Windows services (UAC prompt)

```powershell
# Right-click PowerShell -> Run as administrator
powershell -ExecutionPolicy Bypass -File scripts/install-services.ps1
```

Installs `blog-caddy` and `blog-cloudflared` (LocalSystem, Automatic).
Copies credentials JSON to `%ProgramData%\cloudflared\` and emits
`configs/cloudflared.localsystem.yml` with the path rewritten. Both
services start immediately and verify with `Get-Service`.

## Daily publish flow

```powershell
powershell -ExecutionPolicy Bypass -File scripts/publish.ps1
```

Pipeline:
1. `gen-feeds.ps1` - feed.xml + sitemap.xml (53 posts)
2. `gen-search-index.mjs` - search-index.json (MiniSearch)
3. `uni-run.mjs build` - uni-app H5 build
4. `robocopy /MIR` - mirrors `dist/build/h5/` to `site/`

The two services keep running; Caddy reads `site/` on every request, so
no restart is needed. To force a refresh, `Restart-Service blog-caddy`.

## 4-tier verification

After a publish or any service restart, validate in this order. Each
tier diagnoses a different failure mode; if tier N fails, fix N before
checking N+1.

### Tier A - Caddy is alive (process + port bind)

```powershell
curl.exe -I http://127.0.0.1:48080/        # expect HTTP/1.1 200
```

If "Connection refused": `Get-Service blog-caddy` -> Stopped means the
service died. Check `logs/blog-caddy.err.log` and validate Caddyfile:
`tools\caddy.exe validate --config configs\Caddyfile`.

### Tier B - site/ is populated (publish.ps1 ran)

```powershell
curl.exe -I http://127.0.0.1:48080/static/feed.xml
curl.exe -I http://127.0.0.1:48080/static/sitemap.xml
curl.exe -I http://127.0.0.1:48080/static/search-index.json
```

All three must return 200. If 404: re-run `scripts/publish.ps1`.

### Tier C - SPA routing + browser end-to-end

```powershell
powershell -ExecutionPolicy Bypass -File scripts/diagnose-prod.ps1
```

Boots a fresh Caddy, runs 17 Playwright probes (home cards, post detail
+ refresh, theme cycle, search, tag page, Chinese-path images, real
404s, RSS validity), tears down. Expect "17/17 passed". Screenshots in
`G:/tmp/blog-diag-prod/`.

### Tier D - Tunnel works (remote access)

```powershell
curl.exe -I https://blog.multilab.cc/      # expect HTTP/2 200 from Cloudflare
```

Or open in a browser on a phone over 4G (off your LAN). If A/B/C pass
but D fails: `Get-Service blog-cloudflared`, then check
`logs/blog-cloudflared.err.log` for ingress errors or stale credentials.

## Service management

```powershell
# Status
Get-Service blog-caddy, blog-cloudflared

# Stop / start / restart
Stop-Service    blog-caddy
Start-Service   blog-caddy
Restart-Service blog-cloudflared

# Logs (NSSM redirects stdout/stderr; rotate at 10 MB)
Get-Content logs\blog-caddy.out.log        -Tail 50 -Wait
Get-Content logs\blog-cloudflared.err.log  -Tail 50

# Caddy access log (separate from NSSM stdout, JSON format)
Get-Content logs\blog-caddy-access.log     -Tail 20
```

### Restart-after-reboot validation

Run this after every infrastructure change (T18 acceptance):

1. Shut down (full power off, not sleep).
2. Power on. **Do not log in.** Wait 60 seconds.
3. From a phone on 4G (or another machine on a different LAN), open
   `https://blog.multilab.cc/`.
4. Site should render. If it doesn't:
   - Log in once, then check `Get-Service blog-caddy, blog-cloudflared`
     - both must be Running.
   - Check Event Viewer (`eventvwr.msc`) -> Windows Logs -> System for
     service start failures.
   - Check `logs/blog-*.err.log` for tunnel auth errors.

## Troubleshooting cheatsheet

Mapped to SPEC §9 risks where applicable.

| Symptom | Likely cause | Fix |
|---|---|---|
| `curl -I /` "Connection refused" | Caddy not running | `Get-Service blog-caddy`; if stopped, `Start-Service blog-caddy` and inspect `logs/blog-caddy.err.log` |
| `curl -I /` returns 400 "HTTP request to HTTPS server" | Caddyfile site address missing `http://` scheme | Already fixed in committed Caddyfile; check you haven't reverted line 1 of the site block |
| `/static/feed.xml` returns 404 | publish.ps1 didn't run | Re-run `scripts/publish.ps1` |
| `/post/12` returns 404 | SPA history fallback broken (SPEC §9 row 2) | Caddyfile's catch-all `handle { try_files {path} /index.html }` block missing or out of order |
| `/static/missing-foo.png` returns 200 with HTML body | SPA fallback caught a missing-asset request (T14 trap) | Caddyfile must have separate `handle /static/*` and `handle /assets/*` blocks before the catch-all |
| Chinese-path image 404 | URL not encoded (SPEC §9 row 3) | Re-run import script; `composables/url-encode.ts` must run in markdown image renderer |
| `https://blog.multilab.cc/` returns 502 | cloudflared origin EOF (SPEC §9 row 4) | ingress in cloudflared.yml must point at `http://127.0.0.1:48080` (not `localhost`); Caddy must bind `127.0.0.1:48080` (not `:48080`) |
| Service starts then dies after a few seconds | LocalSystem can't read credentials (SPEC §9 row 7) | Re-run `scripts/install-services.ps1`; it copies the JSON to `%ProgramData%\cloudflared\` and rewrites the path |
| `nssm install` fails with "Access denied" | Not elevated | Right-click PowerShell -> Run as administrator |
| Caddy log says "address already in use" on 48080 | Another process bound the port | `netstat -ano \| findstr :48080`; kill stale `caddy.exe` (`taskkill /F /IM caddy.exe`) |
| Vite/preview origin EOF in dev | allowedHosts missing | Add `blog.multilab.cc` to `vite.config.ts` server.allowedHosts + preview.allowedHosts |
| PowerShell script chokes with parse error | Non-ASCII char (SPEC §9 row 5) | Open in editor, save as UTF-8 without BOM after replacing CJK; then `[Parser]::ParseFile` to verify |

## Uninstall flow

```powershell
# Right-click PowerShell -> Run as administrator
powershell -ExecutionPolicy Bypass -File scripts/uninstall-services.ps1

# Optional: remove generated artifacts
Remove-Item -Recurse -Force site, logs, frontend-uniapp\dist

# Optional: revoke the tunnel
tools\cloudflared.exe tunnel delete blog
# Cloudflare DNS CNAME for blog.multilab.cc must be deleted manually
# in the Cloudflare dashboard (or `cloudflared tunnel route dns -d blog`).

# Optional: remove binaries (rarely needed; just delete tools/)
Remove-Item tools\caddy.exe, tools\cloudflared.exe, tools\nssm.exe
```

`uninstall-services.ps1` only removes the service definitions; binaries
and configs are left intact so `install-services.ps1` can put things
back without re-downloading.

```
  docs/
    deployment.md                   # this file
```
